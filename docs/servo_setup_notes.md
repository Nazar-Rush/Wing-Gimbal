# The new working approach was:

1. Install the Python client tools
2. Clone and build pigpio from source
3. Fix shared library linking
4. Start the daemon

Install Steps

1. Install required packages
   sudo apt update
   sudo apt install -y python3-pigpio pigpio-tools git build-essential python3-setuptools
2. Clone the pigpio source
   git clone https://github.com/joan2937/pigpio.git
   cd pigpio
3. Build pigpio
   make
4. Fix Python install issue

During sudo make install, the following error occurred:

ModuleNotFoundError: No module named 'distutils'

This happened because the install script used distutils, which is no longer present in newer Python versions.

To fix it, edit setup.py:

sed -i 's/from distutils.core import setup/from setuptools import setup/' setup.py

Then run:

sudo make install

Shared Library Fix

After install, pigpiod existed, but failed with:

pigpiod: error while loading shared libraries: libpigpio.so.1: cannot open shared object file: No such file or directory

This was fixed by refreshing the shared library cache:

ls -l /usr/local/lib/libpigpio\*
sudo ldconfig
ldconfig -p | grep pigpio
ldd /usr/local/bin/pigpiod | grep pigpio
sudo /usr/local/bin/pigpiod -v

After this, the library linked correctly.

Start the pigpio daemon

Start the daemon with:

sudo pigpiod

You can verify Python can connect to it with:

python3 -c "import pigpio; pi=pigpio.pi(); print(pi.connected); pi.stop()"

If it prints 1 or True, the daemon is running and the Python library is working.

# Make pigpio start on boot

chat:

Since you built pigpio from source, the cleanest way is to make a systemd service for pigpiod and enable it at boot. Upstream pigpio installs pigpiod into /usr/local/bin, and the repo also includes a sample pigpiod.service unit for auto-starting the daemon.

If systemctl cat pigpiod says no such unit, create one manually. Use this exact file, but point ExecStart at /usr/local/bin/pigpiod on your system, because your source install placed the daemon under /usr/local/bin rather than /usr/bin.

sudo nano /etc/systemd/system/pigpiod.service

Paste this:

```
[Unit]
Description=Pigpio daemon
After=network.target

[Service]
Type=forking
PIDFile=/var/run/pigpio.pid
ExecStart=/usr/local/bin/pigpiod
ExecStop=/bin/kill -SIGTERM $MAINPID
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

The upstream sample unit uses Type=forking and a PIDFile, and pigpio’s README says the daemon uses /var/run/pigpio.pid, so that layout matches how pigpiod runs.

Then reload systemd, enable it, and start it:

sudo systemctl daemon-reload
sudo systemctl enable pigpiod
sudo systemctl start pigpiod
sudo systemctl status pigpiod
